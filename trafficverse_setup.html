🚀 Quick Start (Docker)

# 1. Clone the repository
git clone https://github.com/trafficverse-ai/trafficverse
cd trafficverse

# 2. Copy environment variables
cp .env.example .env
# Edit .env with your values (see table below)

# 3. Download YOLOv11x weights
mkdir -p backend/models
wget https://github.com/ultralytics/assets/releases/download/v8.1.0/yolov11x.pt \
     -O backend/models/yolov11x_traffic.pt

# 4. Start all services
docker compose -f deployment/docker-compose.yml up -d

# 5. Open in browser
open http://localhost:3000        # Frontend
open http://localhost:8000/docs   # API docs (Swagger)

🔧 Local Development (No Docker)

# Backend (Python 3.11+, CUDA 12.1)
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend (Node 18+)
cd frontend
npm install
npm run dev

# Database
createdb trafficverse
psql trafficverse < backend/schema.sql

# Redis
redis-server --daemonize yes

🔬 Model Information

  • Base model: YOLOv11x (Ultralytics) — 86.9M params, 194.9 GFLOPs
  • Training data: 12,000 annotated Indian traffic images (BDD100K + custom Bengaluru dataset)
  • Classes: person, car, motorcycle, bus, truck, helmet, no_helmet, seatbelt, no_seatbelt, license_plate
  • Input size: 640×640 (auto-resize with letterboxing)
  • Inference: TensorRT FP16 on NVIDIA T4/A10G — 2.1ms avg latency
  • Tracking: ByteTrack SORT — 0.5ms per frame, 97.2% MOTA on MOT17
  • OCR: PaddleOCR v3 with Indian license plate fine-tuning — 96.2% character accuracy
